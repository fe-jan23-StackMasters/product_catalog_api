Nice store API documentation

This documentation describes the available endpoints of nice-store API and their parameters.

Overview of endpoints

Get products

URL: /products

HTTP Method: GET

Description: Get a list of products with pagination and sorting capabilities.

Query Parameters:

  • perPage (optional) - number of products per page (default 16)
  • page (optional) - page number (default 1)
  • productType (optional) - type of product (default all)
    • phones
    • tablets
    • accessories
  • sortBy (optional) - sorting method (default newest)
    • name - sort by name alphabetically
    • newest - release date new ➡ old
    • oldest - release date old ➡ new
    • lowest - price low ➡ hight
    • highest - price hight ➡ low

Response:

  • pages - total number of pages
  • models - total number of models on the page
  • products - an array of objects with information about the products

Request: /products?page=1&perPage=2&sortBy=oldest&productType=phones

Get new arrivals

URL: /products/new

HTTP Method: GET

Description: Get a list of new arrivals with the ability to set a limit.

Query Parameters:

  • limit (optional) - number of products (default 12)

Response: an array of objects with information about the products

Using example

Request: /products/new?limit=1

HTTP Status Codes

The following HTTP status codes may be returned by the API:

  • 200 OK - The request was successful.
  • 400 Bad Request - The request was invalid or missing required parameters.
  • 404 Not Found - The requested resource was not found.
  • 500 Internal Server Error - An error occurred on the server while processing the request.
